# Playbook Demo Guiada 35 Dias

Modelo: hibrido local-cloud
Objetivo: convertir piloto en contrato anual con evidencia tecnica, comercial y legal.

## Enfoque por perfil de usuario
1. **Docente individual**
   Resultado esperado: cierre a Docente Esencial/Impulso segun uso.
2. **Coordinador academico**
   Resultado esperado: cierre a Coordinacion Esencial/Control/Gestion.
3. **Direccion academica/TI**
   Resultado esperado: cierre a Institucional Esencial/Integral/Multisede/Sector Publico.
4. **Socio de canal**
   Resultado esperado: cierre a Socio Operador/Crecimiento/Marca Blanca/OEM Integrado.

## Fase 1 (Dia 1-5) Diagnostico y setup
- Discovery operativo/academico.
- Instalacion local + portal cloud.
- Baseline de seguridad y cumplimiento.
- Consentimiento de prueba promocional y finalidades de uso de datos.

## Fase 2 (Dia 6-12) Capacitacion
- Flujo completo docente: banco -> plantilla -> PDF -> OMR -> calificacion -> publicacion.
- Capacitacion de perfiles clave.
- Instrumentacion de eventos de uso para medir adopcion real.

## Fase 3 (Dia 13-20) Operacion asistida
- Operacion real en alcance controlado.
- Seguimiento de incidencias y adopcion.
- Captura estructurada de datos comerciales:
  - frecuencia de uso,
  - tiempo de ciclo,
  - tasa de error/retrabajo,
  - uso de funciones comerciales.

## Fase 4 (Dia 21-28) Endurecimiento
- Simulacro ARCO.
- Prueba de continuidad (caida parcial local/cloud).
- Revision de controles y retencion.
- Segmentacion de propuesta por perfil con base en evidencia de uso.

## Fase 5 (Dia 29-35) Cierre comercial
- Entrega de KPI/ROI.
- Comite de cierre.
- Propuesta final por perfil:
  - Docente individual -> Docente Esencial (entrada), Impulso o Rendimiento,
  - Coordinacion -> Coordinacion Esencial, Control o Gestion,
  - Institucional -> Institucional Esencial, Integral, Multisede o Sector Publico,
  - Socio de canal -> Operador, Crecimiento, Marca Blanca u OEM Integrado.

## Criterios de conversion
- Adopcion minima acordada por usuarios activos.
- Tiempo de ciclo evaluativo bajo umbral.
- Error OMR dentro de objetivo.
- Incidentes criticos = 0 en la ventana final.

## Politica de datos en promocion (obligatoria)
- Acceso promocional condicionado a consentimiento informado.
- Capturar el maximo de datos utiles para producto y conversion, dentro de base legal aplicable.
- Prohibido uso fuera de finalidades declaradas en aviso y contrato de prueba.
- Debe existir ruta de revocacion de consentimiento y ejercicio ARCO/DSR.

## Regla economica del cierre
- Toda oferta final debe validarse contra `CHECKLIST_APROBACION_PRECIO_NIVEL.md`.
- Ningun cierre comercial puede romper margen bruto minimo del 60%.
